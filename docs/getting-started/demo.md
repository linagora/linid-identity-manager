## 🧭 What to Explore

Once logged in, here are the key areas you should explore:

---

### 👤 User Management

* Assign the user to an Organizational Unit
* View user details and attributes

---

### 🏢 Organizational Units

* Browse existing Organizational Units
* Create a new OU
* Understand how users are grouped

---

### 📦 Applications

* View registered applications
* Explore application configuration
* Check defined roles

---

### 🏷️ Roles and Permissions

* See how roles are defined per application
* Understand how permissions are structured

---

### 🔐 Authentication

* Explore OIDC configuration (if visible)
* Understand how authentication is linked to applications

---

### 🧠 Policy Engine (OPA)

* View policy definitions (if exposed)
* Understand how roles are dynamically assigned

---

## 🧪 Example Scenario

Try the following workflow:

1. Assign the user to an Organizational Unit
2. Select an application
3. Check which roles are assigned
4. Modify or simulate policy behavior (if available)

This will give you a good understanding of how LinId manages access.
